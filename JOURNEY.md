## 🧠 0. CORE MODEL
Every journey follows:
**ENTRY → ORIENTATION → VALIDATION → EXPLORATION → DECISION → ACTION → LOOP**

- **Evaluator:** Click → Scan → Understand → Validate → Explore → Decide → Act
- **Owner:** Think → Capture → Curate → Publish → Improve
- **System:** Collect → Filter → Structure → Connect → Display → Adapt

- **Action:** Evaluator pastes JD/Brief.

- **Touchpoints:** [Book Systems Audit Call] button.

### 🔁 STAGE 7 — LOOP

## 🧠 3. MEMORY ENGINE (THE CORE DIFFERENTIATOR)
**Memory = Capture → Structure → Link → Retrieve → Evolve**

### 🔷 Memory Types
- **RAW (HOLD):** Unfiltered, messy capture.
- **STRUCTURED (SIGNALS):** Cleaned, reusable insights.
- **META (PATTERNS):** Recurring truths across domains.
- **APPLIED (SYSTEMS):** Real-world execution (Bankco, NTUC, etc.).
- **DIRECTIONAL:** Current focus (AI, Decision Architecture).

### ⚙️ Critical Rules
1. **FORCED LINKING:** Every signal MUST connect to at least 1 system OR 1 pattern.
2. **PATTERN PROMOTION:** If a theme appears 3+ times, it MUST become a pattern.
3. **DECAY / PRUNING:** Irrelevant/old data is auto-shadowed.
4. **RETRIEVAL-FIRST:** Design for asking: “Where have I solved X?”
5. **REINFORCEMENT:** Reusing an idea strengthens its node.
